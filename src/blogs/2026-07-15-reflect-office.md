The game started at the start of 2024 as a small game jam project. I just wanted to take a break from my current
project at the time and work on something else. I was surprised that my simple typing game got second place at that
time, but didn't think much of it and continued on my project afterwards. I focused on finishing my first steam
game on March 2024 which wasn't anything spectacular.

After the end of a project, there is the usual "what's next?" question coming up. So there has been a longer break
where I've been thinking and maybe even trying out new ideas, I don't exactly remember. But I don't think it has been
that long because around the mid of the year I already decided to circled back to the game, determined to ship 
something small. Initially I just played to add some stats and leaderboard to it, thinking these features would be
good enough for the game.

Here's where things got complicated: I had no clear goal, no plan.

I wasn't sure what the game was about. It had no insentives to continue playing, no progression, no goals to reach
except typing faster and faster. Which is why I got slightly demotivated on continue working on it. I was thinking
of many things I could add to improve it, but wasn't sure what would be good.

I was stuck. No plan. No goal. Just... going through the motions.

# Next Fest

Next Fest was approaching around October and thought it would be a good opportunity to put my game on it, gather some
feedback. This was probably the best decision I've made. The feedback I received was invaluable. It motivated me and 
made me believe in the project again. But it also triggered something dangerous: scope creep.

I kept telling myself, "Just one more feature. Just polish this one thing." Before I knew it, my "small game" had 
grown far larger than I'd ever intended. If I've continue, I would have never finished this game probably. I had to 
make a conscious decision. I set a fixed release date and sticked to it.

That became my finish line. I listed all the features I still planned to make and decided which are the most important
and doable until that date. Everything else went into the trash can. As with most things, to get something done you
can only balance three things: the scope, the time and the resources available. As a solo dev I am the only resource
available and with a fixed release date, I can only reduce the scope to deliver it on time. So that's what I did.

I don't release a game "when it's perfect." or "when it has every feature I can imagine." But when I decide it's done.
If I don't decide that, it might never get finished.

# Local Indie Area

I've also tried showing the game at a local indie event, and the reception was... muted. I guess the game is a bit 
too niche for the general audience of these events. It wasn't because of me being an introvert and much advertising it
or anything...probably. Talking to the other developers there pictured the same image. They purposely show only
specific types of games on these events that have a broader appeal, titles that matched the event's demographic.

It was a valuable lesson: know your venue as much as you know your game. Without a game that is appealing to a broader
audience. It doesn't really make sense to show them at these events.

# Verdict

Overall I'm happy how the game turned out. But there is still the thought in the back of my head about how this game
would have turned out when I had clear target audience in mind. I've tuned this game to be as appealing as possible
for as many people as possible. Not just for the hardcore typist out there, but also for the casual ones.

I guess that will always be the dilemma when developing games. You will never know how a game would have turned out
if you have decided on a different route. So I want to focus more on the following things for my next games

- **Explore Different Directions Early**
Instead of committing to one path and feeling trapped later, I want to prototype multiple approaches in the beginning.
Find the fun quickly, then build around it.

- **Focus on a Target Audience**
No more trying to be everything to everyone. I need to know who I'm building for and make design decisions that serve 
them, not some imaginary "everyone."

- **More Playtesting**
I need to put my work in front of players as early as possible. The feedback will shape the game in ways I can't
anticipate alone.
